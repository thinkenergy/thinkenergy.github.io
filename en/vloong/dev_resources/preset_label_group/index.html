预置标签组

预置标签组简介​

「伏龙平台」为数据集提供了关联标签组功能，同时为部分预置数据集提供了预置标签组。 数据标签是构建高质量数据集的必由之路，平台将不断为预置数据集提供各类预置标签组，构建更大规模更高质量数据集。

车端数据集​

MOVNOP 标签组​

本标签组为与 「MOVNOP 数据集」关联的异常检测标签组，具体信息如下：

  1. 标签组类型：异常检测
  2. 标签组关联：MOVNOP 数据集
  3. 标签组数量：145，与 MOVNOP 数据集车辆数一致
  4. 标签组说明：标签组标签由两个 list 组成的字典，内容为
    {
        'vin': ['xxx', ..., 'xxx'],
        'label': ['0', ..., '1']
    }
    其中 'vin' 中 'xxx' 为 8 位数字编码，与数据集中车辆标识对应；'label' 中 '0' 代表正常车，'1' 代表异常车。
  5. 标签组使用：在 Notebook 环境中，标签组存储于 label 文件夹下；创建训练任务时，可在算法中通过解析 '--label_url' 获取标签组存储的文件夹目录，标签组存储格式为 json 格式。
注意

异常车的编码为

['35030200', '19838988', '71605243','63350557','81048086'，'72694973', '82602335', '33080614', '28105763','87025142', '10177948', '83126407','31367780','85714257', '46996081']

如希望仅针对特征异常车进行分析，也可直接通过上述编码索引异常车。
